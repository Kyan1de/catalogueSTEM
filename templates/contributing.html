Contributing

Okay, some things you'll need to know before getting started:

  1. The backend of this was built using Python + flask + misc. extensions, which are:
    1. flask-sqlalchemy [DB]
    2. tabulate [CLI]
    3. manyterm [CLI]
  2. The frontend is plain html + css + javascript

while you can use another language to add to the project, I wouldn't recommend doing so as it makes the project harder to maintain.

The code is currently in one* file, split into four sections.

*the flaskLogger.py file is a workaround for an issue with manyterm, you can pretty safely ignore it

Section 1: Initialization

This section Initializes and configures everything prior to the server starting. This includes any tables you may have added to the project

Section 2: Routing

This section sets up all of the pages accessible to client-side users. Please note, this is only really necessary for pages that have more advanced behavior, static assets such as images and css files should just be put into the static folder.

Section 3: the CLI

This section holds all of the functions for the command-line interface. Adding to this should be fairly easy, only requiring you to add to the command and help tables below all of the functions. Documentation for how those tables behave will be found just above them.

Section 4: Entrypoints

This section handles all of the startup logic. currently it always starts the flask server as a subprocess, sending its output to a different window, and starts the command-line input loop. If any other modes for the application are made, you will likely need to make changes to this section.

Possible additions

  • A way to import data from google sheets, likely using TSV format as an inbetween
